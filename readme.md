Bestell App – User Guide

Overview

    This project is a front-end training clone of a Lieferando-style food ordering page. It is designed to simulate the user experience of browsing a restaurant menu, adding items to a basket, updating quantities, and completing a sample checkout flow.
    The page is interactive, but it is not a real delivery platform. It is intended for demonstration and learning purposes only.

What You Can Do

    Users can interact with the visible elements on the page and test the basic ordering flow.
        - Browse the menu of the restaurant shown on the page.
        - Add dishes to the basket.
        - Increase, reduce, or remove items in the basket.
        - See prices update automatically.
        - See the basket summary update based on the selected items.
        - Perform an example order.

How the Basket Works

    The basket is connected to the menu data used in the project. When a dish is added, the item appears in the basket and the price calculation is updated automatically.
    You can use the basket controls to adjust the quantity of each item. The total price and basket overview react to these changes in the front end.

Example Order

    The project includes an example order action to demonstrate the final step of a checkout flow.
    When this action is triggered:
        - the basket is cleared,
        - the order is treated as completed inside the interface,
        - but no real order is sent anywhere.
    This means the feature is only a visual and functional front-end simulation.

Current Limitations

    This project intentionally focuses on the front end and does not include real platform functionality.
    The following features are not available:
        - Creating user profiles or customer accounts.
        - Logging in or registering.
        - Viewing real past or current orders.
        - Sending data to a server.
        - Submitting real orders to a restaurant.
        - Using multiple restaurants.
    Only the single restaurant shown in the interface is part of this project.

Important Note for Users

    Please use this page as a demonstration prototype, not as a real ordering service. All interactions happen only inside the browser interface.
    Even though the page behaves like a delivery website in many places, no personal data, order data, or payment data is processed or transmitted.

Intended Purpose

    This clone was developed for training and educational purposes. Its goal is to demonstrate front-end interaction, interface behavior, basket logic, and dynamic rendering based on stored menu data.
    It is best understood as an interactive sample project that shows how a food ordering interface can work visually and functionally on the client side.
